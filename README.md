# RK CRM

نظام متابعة عملاء لـ RK Real Estate — حسابات منفصلة لكل سيلز، قاعدة بيانات حقيقية، ولوحة تقارير مجمعة للأدمن.

---

## 1. إنشاء حساب Supabase (مجاني)

1. روح على [supabase.com](https://supabase.com) واعمل حساب مجاني.
2. اعمل **New Project**:
   - اختار اسم للمشروع (مثلاً `rk-crm`)
   - حدد باسوورد لقاعدة البيانات (احفظه في مكان آمن)
   - اختار أقرب Region (Frankfurt أو أي منطقة قريبة من مصر)
3. هتنتظر دقيقة أو اتنين لحد ما المشروع يخلص setup.

**التكلفة:** الخطة المجانية (Free tier) كافية جدًا لـ 4-10 سيلز — فيها 500MB قاعدة بيانات و 50,000 مستخدم شهريًا، وانتم بعيدين جدًا عن الحد ده.

---

## 2. إنشاء الجداول (Database Schema)

1. من القائمة الجانبية في Supabase، افتح **SQL Editor**.
2. افتح ملف `schema.sql` الموجود في هذا المشروع، انسخ **كل المحتوى**.
3. حطه في SQL Editor ودوس **Run**.
4. ده هيعمل: جدول العملاء، جدول المتابعات، جدول الأهداف، جدول البروفايلات، وصلاحيات الأمان (RLS) اللي تخلي كل سيلز يشوف بياناته بس، والأدمن يشوف الكل.

---

## 3. ربط المفاتيح (API Keys)

1. من القائمة الجانبية: **Project Settings > API**.
2. هتلاقي:
   - **Project URL**
   - **anon public key**
3. في مجلد المشروع، اعمل نسخة من ملف `.env.example` وسمّيها `.env`:
   ```
   cp .env.example .env
   ```
4. افتح `.env` وحط القيم بتاعتك:
   ```
   VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJxxxxxxxxxxxxxxxxxxxx
   ```

---

## 4. تشغيل المشروع محليًا (لاختبار على اللابتوب)

محتاج تثبت [Node.js](https://nodejs.org) (أي نسخة 18 أو أحدث) لو معندكش.

```bash
npm install
npm run dev
```

هيفتح على `http://localhost:5173` — تقدر تفتحه من المتصفح وتجرب.

---

## 5. نشر الموقع (عشان كل سيلز يدخل من تليفونه/لابتوبه)

أسهل طريقة مجانية: **Vercel**

1. اعمل حساب على [vercel.com](https://vercel.com) (تقدر تربطه بـ GitHub).
2. ارفع المشروع على GitHub (repo جديد).
3. من Vercel: **New Project** → اختار الـ repo.
4. في خطوة Environment Variables، ضيف:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   (بنفس القيم اللي في ملف `.env` بتاعك)
5. دوس Deploy.

هتاخد لينك (مثلاً `rk-crm.vercel.app`) — ده اللينك اللي يفتحه كل سيلز من أي جهاز.

---

## 6. إنشاء الحسابات

1. الأدمن (انت) يفتح الموقع ويعمل حساب جديد (Sign up) بالاسم والإيميل والباسوورد.
2. روح تاني لـ Supabase > SQL Editor، ونفّذ السطر ده (غيّر الإيميل بإيميلك):
   ```sql
   update public.profiles set role = 'admin'
   where id = (select id from auth.users where email = 'your-email@example.com');
   ```
3. سجل خروج ودخول تاني — هتلاقي تبويب "تقارير الفريق" ظهر.
4. كل سيلز يفتح نفس اللينك ويعمل حساب لنفسه (Sign up) — حسابه يبقى تلقائيًا "sales" ويشوف بياناته بس.

---

## النظام بيشمل

- **لوحة التحكم**: عملاء نشطين، متابعات مستحقة، صفقات الشهر، معدل التحويل، قمع المبيعات.
- **العملاء**: كانبان بالمراحل (عميل جديد → تم التواصل → اجتماع → تفاوض → بيع/ملغي)، بحث، بيانات كل عميل.
- **المتابعات**: متأخرة / اليوم / قادمة.
- **الأهداف الشهرية**: هدف صفقات واجتماعات + تتبع التقدم.
- **تقارير الفريق (أدمن فقط)**: قمع مبيعات للفريق كله + جدول أداء كل سيلز.

كل سيلز يشوف بياناته فقط (مفروض من قاعدة البيانات نفسها عبر RLS). الأدمن يشوف بيانات الكل + التقارير المجمعة.

## ملاحظات

- لو حد من السيلز نسي الباسوورد، Supabase فيها صفحة "Forgot password" تلقائية — ممكن نفعّلها لو احتجتها بعدين.
- لو عايز تغيّر اسم أي سيلز بعد التسجيل، تقدر تعدله مباشرة من Supabase > Table Editor > profiles.
